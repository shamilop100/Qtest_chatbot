Problems & Solutions
1
System prompt sent on EVERY message
assistant_prompt.txt · ~120 tokens × every API call
🔴 Biggest Waste
❌ What's happening
client.messages.create( model=MODEL_NAME, system=SYSTEM_PROMPT, # ← full prompt max_tokens=250, # re-sent every messages=[{...}] # single call )
✅ Fix — use Prompt Caching
client.messages.create( model=MODEL_NAME, system=[{ "type": "text", "text": SYSTEM_PROMPT, "cache_control": { "type": "ephemeral" # ← cached! } }], max_tokens=250, messages=[{...}] )
Why it matters: Anthropic's Prompt Caching charges only 10% of normal input cost for cached tokens after the first call. Your system prompt is ~120 tokens. At 10,000 messages/month that's ~1.2M tokens — caching saves 90% of that cost. One line change, massive saving.
2
max_tokens=250 is too high for a 60-word reply
claude_service.py · you're paying to reserve tokens you never use
🔴 Direct Waste
❌ Problem
max_tokens=250 # Your prompt says reply # must be under 60 words # (~80 tokens max). # You're reserving 250 — # 3x more than needed.
✅ Fix
max_tokens=120 # INTENT line ≈ 15 tokens # REPLY 60 words ≈ 80 tokens # Safety buffer ≈ 25 tokens # Total needed: ≈ 120 max # Saves ~50% output budget
Why it matters: max_tokens doesn't directly cost money — you pay for tokens generated, not reserved. But setting it accurately prevents Claude from rambling beyond your 60-word limit, keeping output consistently short and cheap. Combined with temperature=0.3, Claude stays tight and on-format.
3
No cache for repeated questions
main.py · "fee ethra?" asked 100x → Claude called 100x
🟠 High Saving
❌ Problem
# Every message hits Claude API # Students ask the same things: # "fee ethra?" # "online aano offline?" # "certificate kittuo?" # Each → 1 API call → billed
✅ Fix — in-memory reply cache
import hashlib _cache = {} def ask_claude(message, hint): key = hashlib.md5( f"{message}{hint}".encode() ).hexdigest() if key in _cache: return _cache[key] # free! result = _call_claude(...) _cache[key] = result return result
Why it matters: Training institute students ask the same 10–15 questions repeatedly. Fee, duration, online/offline, placement, demo — these repeat constantly. Caching identical queries means zero API cost for repeat questions. For a busy day with 200 messages, easily 40–60% could be cache hits.
4
Sending unnecessary context in the prompt
claude_service.py · extra words = extra tokens billed
🟠 Medium Saving
❌ Verbose prompt content
content = f""" Customer message: {message} LOCAL_INTENT_HINT: {hint} Note: The hint may be wrong. Trust the message, not the hint. """ # ~20 extra tokens per call # = 200,000 extra tokens/10K msgs
✅ Trim to minimum
content = ( f"MSG: {message}\n" f"HINT: {hint}" ) # Claude already knows from # system prompt what HINT means. # No need to repeat instructions.
Why it matters: Your system prompt already explains what LOCAL_INTENT_HINT means and that it may be wrong. Repeating "Note: hint may be wrong" in every user message adds ~15 tokens per call. At 10K messages/month = 150,000 wasted tokens.
5
Using Claude for greetings ("hi", "hello")
main.py · simple greetings don't need an API call at all
🔵 Easy Win
❌ Every message hits Claude
user = input("User: ") lang, hint = get_hint(user) reply = ask_claude(user, hint) # "hi" → full API call 💸
✅ Short-circuit known intents
STATIC_REPLIES = { "greeting": "നമസ്കാരം! എങ്ങനെ സഹായിക്കാം?", } lang, hint = get_hint(user) if hint in STATIC_REPLIES: print(STATIC_REPLIES[hint]) else: intent, reply = ask_claude( user, hint )
Why it matters: "Hi", "Hello", "Namaskaram" — these are templated responses. No reasoning needed. Skip Claude entirely and reply instantly for free. You can expand STATIC_REPLIES to cover any high-confidence, fixed-answer intents your institute has.
