# Problem ❌ Before ✅ After Impact
1
Model loads on import
pipeline() runs at module load time — 5–6 second freeze before the app does anything
lang_detector = pipeline( "text-classification", model="papluca/..." ) # Runs IMMEDIATELY on import
_lang_detector = None def _get_lang_detector(): global _lang_detector if _lang_detector is None: _lang_detector = pipeline(...) return _lang_detector # Loads only on first message
 🔴 High
App starts instantly. Model loads once, cached forever.
2
Only 4 of 9 intents detected
syllabus, placement, certificate, demo, greeting all fell through to "other" — useless hint to Claude
if contains(t, PRICING_WORDS): hint = "pricing" elif contains(t, DURATION_WORDS): hint = "duration" elif contains(t, ONLINE_WORDS): hint = "online_offline" elif contains(t, LOCATION_WORDS): hint = "location" else: hint = "other" # ← 5 intents always land here
_INTENT_MAP = [ ("greeting", GREETING_WORDS), ("pricing", PRICING_WORDS), ("duration", DURATION_WORDS), ("syllabus", SYLLABUS_WORDS), ("placement", PLACEMENT_WORDS), ("certificate", CERTIFICATE_WORDS), ("demo", DEMO_WORDS), ("online_offline",ONLINE_WORDS), ("location", LOCATION_WORDS), ] # All 9 intents covered
 🔴 High
Claude now gets useful hints for all query types.
3
Tiny keyword lists miss real Manglish
"enthanu fee", "ethra naal", "job kittuo" — common student phrases not matched at all
PRICING_WORDS = [ "ഫീസ്","വില", "charge","rate","amount" ] # Only 5 words
PRICING_WORDS = [ "ഫീസ്","വില","charge","rate", "amount","fee","fees","price", "cost","paisa","rupee","rs", "money","ethra aanu","how much", "enthanu","എത്ര രൂപ" ] # 14+ real-world phrases
 🔴 High
Matches real student input in Malayalam, English, Manglish.
4
contains() uses slow manual loop
For loop with early return works, but is verbose and not Pythonic
def contains(text, keywords): for w in keywords: if w in text: return True return False
def contains(text, keywords): return any( w in text for w in keywords )
 🟢 Low
Cleaner, shorter, same performance.
claude_service.py
claude_service.py Claude API · Response Handling
# Problem ❌ Before ✅ After Impact
5
No temperature set → inconsistent format
Default temp=1.0 makes Claude vary its output format randomly, breaking INTENT/REPLY parsing
response = client.messages.create( model=MODEL_NAME, system=SYSTEM_PROMPT, max_tokens=250, messages=[...] ) # temperature defaults to 1.0
response = client.messages.create( model=MODEL_NAME, system=SYSTEM_PROMPT, max_tokens=250, temperature=0.3, # ← structured output messages=[...] )
 🔴 High
Claude follows INTENT/REPLY format consistently every time.
6
Zero error handling — any failure crashes the loop
Rate limit, timeout, or connection drop kills main.py entirely with an unhandled exception
response = client.messages.create(...) return response.content[0].text # API error → entire app crashes 💥
try: response = client.messages.create(...) return response.content[0].text except anthropic.RateLimitError: time.sleep(delay) # retry except anthropic.APIConnectionError: time.sleep(delay) # retry except anthropic.APIStatusError: break # don't retry 4xx # Returns FALLBACK_REPLY on failure
 🔴 High
App stays alive. User gets polite Malayalam error message instead of crash.
7
Raw text returned — INTENT/REPLY shown to user
main.py prints the full raw Claude response including "INTENT: pricing\nREPLY:" labels
return response.content[0].text # main.py prints: # INTENT: pricing # REPLY: ഞങ്ങളുടെ കോഴ്സ്...
def parse_response(raw): for line in raw.splitlines(): if line.startswith("INTENT:"): intent = line.split(":",1)[1].strip() elif line.startswith("REPLY:"): reply = line.split(":",1)[1].strip() return intent, reply # main.py prints only: # ഞങ്ങളുടെ കോഴ്സ്...
 🟠 Med
Clean output for users. Intent available separately for logging/analytics.
8
Prompt doesn't tell Claude to distrust the hint
Claude sometimes over-weights LOCAL_INTENT_HINT even when it's wrong, giving wrong replies
content = f""" Customer message: {message} LOCAL_INTENT_HINT: {hint} """
content = ( f"Customer message: {message}\n" f"LOCAL_INTENT_HINT: {hint}\n" f"Note: The hint may be wrong. " f"Trust the message, not the hint." )
 🟠 Med
Claude's own reasoning wins over a bad local hint → better intent accuracy.
9
File opened without closing (no `with` block)
open() without with leaves the file handle unclosed — bad practice, can cause issues on some systems
SYSTEM_PROMPT = open( "assistant_prompt.txt", encoding="utf-8" ).read() # File handle never closed
with open("assistant_prompt.txt", encoding="utf-8") as f: SYSTEM_PROMPT = f.read() # Auto-closed after read ✓
 🟢 Low
Correct resource management. Raises clear error if file missing.
