Step-by-Step Migration
1
Dockerise hf_local.py as a microservice
Wrap your HuggingFace model in a FastAPI app and a Dockerfile. This becomes your ECS container — model stays warm in memory permanently, no cold starts.
# hf_service/app.py from fastapi import FastAPI from hf_local import get_hint app = FastAPI() @app.post("/hint") def hint(payload: dict): lang, hint = get_hint(payload["text"]) return {"lang": lang, "hint": hint} # Dockerfile FROM python:3.11-slim COPY . /app WORKDIR /app RUN pip install fastapi uvicorn transformers torch CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]
ECS Fargate Docker Fixes cold start
2
Push Docker image to ECR
Amazon ECR is AWS's private Docker registry. Push your image here — ECS will pull from it automatically on deploy.
# Create ECR repo aws ecr create-repository --repository-name malayalam-hf-service # Authenticate, build, tag, push aws ecr get-login-password | docker login --username AWS \ --password-stdin <account>.dkr.ecr.ap-south-1.amazonaws.com docker build -t malayalam-hf-service . docker tag malayalam-hf-service:latest \ <account>.dkr.ecr.ap-south-1.amazonaws.com/malayalam-hf-service:latest docker push <account>.dkr.ecr.ap-south-1.amazonaws.com/malayalam-hf-service:latest
ECR ap-south-1 (Mumbai)
3
Store API key in Secrets Manager
Remove CLAUDE_API_KEY from .env forever. Store it in AWS Secrets Manager — Lambda fetches it at runtime via IAM role. No key in code, no key in environment variables.
# Store the secret aws secretsmanager create-secret \ --name "prod/chatbot/claude-api-key" \ --secret-string '{"CLAUDE_API_KEY":"sk-ant-..."}' # In config.py — replace dotenv with: import boto3, json def get_secret(): client = boto3.client("secretsmanager", region_name="ap-south-1") secret = client.get_secret_value(SecretId="prod/chatbot/claude-api-key") return json.loads(secret["SecretString"])["CLAUDE_API_KEY"] CLAUDE_KEY = get_secret()
Secrets Manager Zero key exposure
4
Convert main.py loop → Lambda handler
Replace the while True loop with a Lambda handler function. API Gateway calls this for every incoming message. Includes Redis cache check before hitting Claude.
# lambda_handler.py import json, hashlib, redis from hf_client import get_hint_remote # calls ECS microservice from claude_service import ask_claude redis_client = redis.Redis(host=REDIS_HOST, port=6379, decode_responses=True) def handler(event, context): body = json.loads(event["body"]) message = body["message"] # 1. Check cache first cache_key = hashlib.md5(message.encode()).hexdigest() cached = redis_client.get(cache_key) if cached: return {"statusCode": 200, "body": cached} # free reply ⚡ # 2. Get intent hint from ECS lang, hint = get_hint_remote(message) # 3. Call Claude intent, reply = ask_claude(message, hint) # 4. Cache result for 24h redis_client.setex(cache_key, 86400, json.dumps({"intent": intent, "reply": reply})) return {"statusCode": 200, "body": json.dumps({"reply": reply})}
Lambda Redis Cache Replaces main.py
5
Create API Gateway + Wire Everything
Create a POST /chat endpoint in API Gateway. Enable throttling (e.g. 100 req/sec max) to protect from abuse and cost spikes. Point it at your Lambda.
# Using AWS CLI aws apigatewayv2 create-api \ --name "MalayalamChatbotAPI" \ --protocol-type HTTP \ --target arn:aws:lambda:ap-south-1:<account>:function:chatbot-handler # Add throttling to prevent bill spikes aws apigatewayv2 create-stage \ --api-id <api-id> \ --stage-name prod \ --default-route-settings \ ThrottlingBurstLimit=50,ThrottlingRateLimit=20
